按面试真实考察结构去系统准备。C++ 面试其实稳定分成几块：语言机制、内存与对象模型、STL/复杂度、并发、工程与性能。你要做的是把这些点变成“能解释+能举例+能写代码”的组合，而不是零散记忆。

先说整体方法：把知识分成模块，每个模块都做到三件事——能讲清概念本质、能解释为什么这样设计、能写一段最小代码证明理解。你准备时不要只记定义，一定要能回答“什么时候会出问题”“和其他方案对比如何”“底层发生了什么”。这是区分会背和真正理解的关键。

第一块是语言与对象模型，这是最核心的。你必须非常清楚对象生命周期（构造、拷贝、移动、析构）、RAII 的意义、拷贝构造和移动构造的区别、什么时候会触发移动语义、为什么要有 rule of 3/5/0。这里面试官最喜欢问的是“你写的类会不会资源泄漏”“vector 扩容时发生了什么”“为什么 move 能提升性能”。你要能从“内存 + 拷贝成本 + 所有权转移”三个角度解释。

第二块是内存管理。包括 stack vs heap、new/delete vs malloc/free 的区别、智能指针（unique_ptr、shared_ptr、weak_ptr）的语义和实现原理、shared_ptr 的引用计数问题、循环引用怎么解决。重点不是记 API，而是理解 ownership model：谁拥有资源、什么时候释放、有没有 double free 或 leak 风险。面试官很常问“shared_ptr 为什么不是线程安全的”“weak_ptr 的意义是什么”。

第三块是 STL 和数据结构。你要清楚 vector、deque、list、map、unordered_map 的底层结构、复杂度和使用场景。比如 vector 为什么扩容是 2 倍、deque 为什么适合头插、map 为什么是红黑树、unordered_map 什么时候退化。这里建议你用“复杂度 + 内存布局 + cache 友好性”来统一解释，否则容易碎。很多人只会说 O(log n)，但面试官更想听你说为什么以及在工程里怎么选。

第四块是模板和泛型。至少要理解模板是编译期机制、模板实例化、函数模板 vs 类模板、typename/decltype、SFINAE 或 concepts 的基本思想。这里不要求你写很复杂的 metaprogramming，但要能解释“为什么模板能做到零开销抽象”，以及“和虚函数的 trade-off 是什么”。

第五块是多态与虚函数。你要清楚静态多态（模板/CRTP）和动态多态（virtual）的区别，虚函数表大概怎么实现，override/final 的作用，什么时候用虚函数、什么时候不用。常见问题是“虚函数为什么有开销”“析构函数为什么要 virtual”“多重继承有什么坑”。

第六块是并发（现在几乎必问）。至少要掌握线程创建、mutex、lock_guard、unique_lock、condition_variable 的基本用法，以及死锁产生的条件。进阶一点要理解内存模型、atomic、数据竞争、false sharing。面试官很喜欢问“这个代码有没有 data race”“怎么设计线程安全结构”。这里不要背 API，要能分析并发场景。

第七块是性能与工程。包括拷贝开销、move 优化、cache locality、inline、编译期 vs 运行期、多线程开销。你要能解释“为什么 vector 比 list 快（很多场景下）”“为什么 small object optimization 有用”。这部分是很多人忽略，但区分度很高。

准备方法上，建议你按模块建立一个“知识卡片体系”，每个点写三行：一句话本质、一段典型代码、一个常见坑。然后每个模块找 10–20 个高频问题，自己用口述方式练到可以 2 分钟讲清楚。不要只看，要“输出”。如果你讲不顺，说明理解还不够。

另外一个非常有效的方法是“反推面试题”：比如 vector、shared_ptr、virtual function 这些核心点，你可以假设自己是面试官，设计问题链条，比如“vector 扩容会发生什么 → 会调用哪些构造函数 → move 和 copy 有什么区别 → 如果元素不可移动会怎样”。这样练几轮，你会发现所有问题其实是连在一起的，而不是零散知识。

最后一点，C++ 面试不是只看八股，一定会结合 coding。你要在写题时刻意用到这些知识，比如用 unique_ptr 管理资源、用 move 避免拷贝、解释你为什么选 vector 而不是 list。能把“语言理解”体现在代码里，才是面试官真正想看的。

一句话总结：不要把 C++ 八股当成要记的点，而是当成一套“解释程序如何在内存和时间上运行”的统一模型来准备。
