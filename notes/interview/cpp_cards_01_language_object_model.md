# C++ 卡片 01：语言与对象模型（50）

1. **对象生命周期**：对象从构造开始，到析构结束，生命周期决定资源何时获取和释放。离开作用域自动析构是 C++ 管理资源的核心机制。
2. **RAII**：资源放进对象里管理，避免手动 `open/close`、`new/delete` 的配对失误。面试中要强调“异常路径也安全”。
3. **默认构造**：类成员如果不在初始化列表显式初始化，就会走默认初始化规则。内置类型若未初始化可能是未定义值。
4. **成员初始化列表**：初始化发生在函数体之前，性能和语义都更正确。`const` 成员和引用成员只能在这里初始化。
5. **初始化顺序**：真实顺序由成员声明顺序决定，不由初始化列表书写顺序决定。顺序写错可能引发隐藏 bug。
6. **拷贝构造**：用于“新对象从旧对象构造”的场景。若类持有资源，必须明确是深拷贝还是共享语义。
7. **拷贝赋值**：用于“已存在对象被覆盖”的场景。要考虑自赋值保护和失败时对象状态是否保持有效。
8. **移动构造**：把资源所有权从源对象转移到新对象，减少深拷贝成本。源对象只需保持“可析构、可重新赋值”。
9. **移动赋值**：先处理目标对象已有资源，再接管源对象资源。实现时要避免自移动导致资源被错误释放。
10. **析构函数**：析构一般应保证不抛异常，否则栈展开期间会触发 `std::terminate`。析构只做清理，不做复杂业务逻辑。
11. **Rule of 3**：如果你手写了析构函数，通常也要考虑拷贝构造和拷贝赋值。因为资源释放与复制语义是绑定的。
12. **Rule of 5**：在 Rule of 3 基础上补上移动构造和移动赋值。现代 C++ 容器优化高度依赖移动语义。
13. **Rule of 0**：优先让成员对象自己管理资源（比如智能指针/容器）。这样你通常不需要手写特殊成员函数。
14. **值语义**：每个对象独立拥有状态，复制后互不影响。接口简单直观，适合大多数业务对象。
15. **引用语义**：多个对象共享同一底层实体，复制成本低但生命周期更难控。要警惕悬垂引用和并发读写问题。
16. **左值与右值**：左值有稳定身份，右值通常是临时值。理解值类别是掌握移动语义的前提。
17. **万能引用**：模板中的 `T&&` 可根据实参折叠成 `T&` 或 `T&&`。它是完美转发的基础。
18. **`std::move`**：它只是一个强制转换，把表达式标记为右值。真正是否移动，取决于目标类型是否有移动操作。
19. **`std::forward`**：在模板转发场景中保留原始值类别。否则右值实参可能被当左值传下去，导致多余拷贝。
20. **返回值优化（RVO）**：编译器常直接在目标位置构造返回对象，减少拷贝/移动。很多场景下无需手工 `std::move` 返回值。
21. **临时对象生命周期延长**：临时对象绑定到 `const T&` 后可延长生命周期。绑定到非常量引用一般不允许。
22. **`explicit`**：用于禁止单参数构造函数的隐式转换。能减少“看似合法但语义错误”的调用。
23. **`=default`**：让编译器生成默认实现，同时明确表达设计意图。比“空函数体”更准确也更安全。
24. **`=delete`**：在编译期明确禁止某操作，如禁止拷贝。比运行时报错更早发现问题。
25. **聚合初始化**：对简单数据结构可一次性列出成员值，减少遗漏。在配置对象和测试数据中很高效。
26. **`const` 成员函数**：承诺不修改对象逻辑状态，提升接口可读性和可复用性。也允许 `const` 对象调用。
27. **`mutable`**：在 `const` 函数里允许修改缓存、统计等“非逻辑状态”。应谨慎使用，避免破坏 `const` 语义信任。
28. **静态成员**：属于类级别，不属于具体对象实例。适合做全体实例共享的计数器或配置。
29. **空类大小**：空类对象通常也占 1 字节，保证不同对象地址可区分。这是语言对象模型的基础规则之一。
30. **内存对齐**：成员布局会受对齐规则影响，导致 padding。对齐能提升访问效率，但会增大对象体积。
31. **平凡类型（Trivial）**：平凡拷贝/析构类型更接近 C 风格内存模型。底层序列化或共享内存场景常依赖这一点。
32. **`this` 指针**：非静态成员函数都有隐式 `this`。理解它有助于解释链式调用和成员访问本质。
33. **友元（friend）**：可访问私有成员，适合紧耦合辅助类或运算符重载。使用过多会破坏封装边界。
34. **浅拷贝风险**：复制裸指针只会复制地址，不复制资源本体。两个对象析构同一资源会造成 double free。
35. **深拷贝代价**：深拷贝更安全但开销更高。现代实践常结合移动语义和共享策略平衡性能。
36. **copy-and-swap**：先复制临时对象再交换，能提供强异常安全保证。实现赋值运算符时很常见。
37. **异常安全等级**：至少保证对象不泄漏且保持可用（基本保证）。更高要求是失败后状态完全不变（强保证）。
38. **对象切片**：派生类对象按值传给基类参数会丢失派生部分。多态场景应传引用或指针。
39. **悬垂引用/指针**：引用了已经释放的对象是高危未定义行为。常见于返回局部变量地址或容器失效迭代器。
40. **placement new**：在指定内存上构造对象，常用于内存池。对应析构必须显式调用，不能直接 `delete`。
41. **构造异常处理**：构造函数抛异常时，已构造完成的成员会自动析构。未构造完成的部分不会被访问。
42. **C++17 临时物化优化**：很多返回临时对象的路径可直接原地构造。要从“值语义”而不是“对象搬运”理解代码。
43. **`noexcept` 与容器行为**：容器扩容时更偏向移动 `noexcept` 类型。否则可能退回拷贝以维持异常安全。
44. **特殊成员生成规则**：你手写某些特殊成员后，编译器可能不再自动生成其他成员。要检查类是否仍满足预期语义。
45. **资源拥有权建模**：设计类时先回答“谁拥有资源、谁释放资源”。这比 API 细节更决定类是否安全。
46. **复制语义一致性**：复制后两个对象都应各自可用、可独立析构。否则接口会在边界条件下崩溃。
47. **类不变量**：对象在任意公开接口调用前后都应满足不变量。这是写出稳定类的关键检查点。
48. **最小暴露原则**：减少可变接口和内部细节暴露，能显著降低误用概率。封装本质是管理复杂度。
49. **按值传参 + 移动接管**：对可拷贝可移动对象，按值接收再 `std::move` 到成员常简洁高效。可统一处理左值和右值调用。
50. **面试表达结构**：先讲定义和语义，再讲成本和 trade-off，最后给一段最小代码。这样回答最容易体现“理解而非背诵”。
